# 🇬🇧 Track: Tech English for DevOps

> Har kuni **5-10 daqiqa**. Maqsad: dokumentatsiya o'qish, xato xabarlarini
> tushunish, intervyu va jamoa bilan ingliz tilida ishlash.

## Nega English?
DevOps'ning 95% materiali ingliz tilida: docs, Stack Overflow, error message,
GitHub issue, intervyu. Texnik ingliz tili — bu **alohida superpower**.

## Har kunlik format (mission ichidagi 🇬🇧 English Drop)
1. **5 ta yangi texnik so'z** + ta'rifi + jumlada ishlatish
2. **1 ta real error message yoki doc parchasi** — tarjima qil va tushuntir
3. **1 ta "speaking" mashqi** — bugun o'rganganingni 2 jumlada ingliz tilida yoz

| Bosqich | Mavzu |
|---------|-------|
| 1 | Filesystem & shell: directory, permission, owner, path, execute |

| 4 | Git: commit, branch, merge, conflict, pull request, rollback |
| 5 | Docker: image, container, build, layer, registry, volume |
| 6 | CI/CD: pipeline, build, deploy, artifact, trigger, stage |
| 7 | Kubernetes: pod, node, cluster, scale, rollout, namespace |
| 8 | Cloud/IaC: provision, infrastructure, instance, region, state |
| 9 | Observability: metric, log, alert, dashboard, threshold |
| 10 | Interview English: "Tell me about...", troubleshooting talk |
